## 決定済み

### wiki の構造

**`/wiki` に置き、既存パスの下に入れなかった。**
`content/database/higher-brain-function/` と扱うものが違う（事典 vs 問い）ため、
階層を混ぜると性質の違いが見えなくなる。相互参照は frontmatter の `refs` で行う。

**`related` は問い同士専用にし、事典への参照は `refs` に分けた。**
同じ配列に混ぜると型が曖昧になり、逆リンク生成で問いと記事を区別できなくなる。

**`id` は `q-001` からの連番。領域プレフィックスを付けなかった。**
問いは領域をまたぐことが多く（`q-004` は失語・病識・メタ認知にまたがる）、
id に意味を持たせると後から分類が変わったときに破綻する。分類は `domain` が担う。

**`q-000` はテンプレート見本。一覧・件数・議題キューから除外している**
（`lib/questions.ts` の `TEMPLATE_ID`）。実際の問いではないので未解決件数に混ぜると
議題が水増しされる。ただし完全に隠すとスキーマ変更時の型崩れに気づけないので、
一覧末尾からのみ辿れるようにした。**削除しないこと。**

**`_inbox/` にあること自体が「未採用」を表す。`status` に `inbox` を作らなかった。**
`status` は `open | mapped | evidenced` の3値。ここに4つ目を足すと wiki 側の
状態遷移が濁る。場所が状態を表す設計にした。

**スタブに `id` を振らない。採用時に採番する。**
未採用のまま番号を消費すると、捨てた問いの分だけ `q-NNN` が飛ぶ。

### lastmod の自動更新 — 一度失敗して設計を変えた

**最終的な方式:** `git status` で変更が検出されたファイルにだけ今日を打刻する。

**当初やろうとして破棄した方式:** git の最終コミット日を全ファイルに引き当てる。
**冪等にならなかった。** 実測で差分が 320 → 329 → 337 件と増え続けた。

原因は2つあり、**2つ目が致命的だった**。

1. `git log --name-only` は既定（`core.quotePath=true`）で非ASCIIパスを
   `\344\270\241...` と8進エスケープする。**全1323パス中1152（87%）が照合不能**だった。
   `content/neurology/` はほぼ日本語ファイル名なのでほぼ全滅。
   → `-c core.quotePath=false` で解決（**この知見は他のスクリプトでも有効**）
2. エスケープを直しても churn が残った。単一パスのログ解析という設計自体が
   不安定だったため、**設計を捨てた**。

**教訓として残す:** 全ファイルの日付を履歴から再構成する必要はなかった。
変更されたファイルだけを対象にすれば構造的に冪等になる。

### 調査の方法論（Phase 2 で確立）

**詳細は `CLAUDE.md` に規約として書いた。ここには「なぜ」を残す。**

**PubMed に自然言語クエリを投げない。** PubMed は入力を AND 分解する。
`language is primarily a tool for communication rather than thought` と投げたところ、
看護用語の標準化・難民健康スクリーニング・医学教育のAI・救急待ち時間アプリ・
GPT-4o翻訳が返り、**目的の論文は1件も含まれなかった**。フィールドタグ付き
（`Fedorenko E[Author] AND Nature[Journal]`）に切り替えたら即座に見つかった。

**PubMed を既定の情報源にしない。** 生物医学限定で哲学・社会科学を含まない。
`q-005` `q-007` `q-008` の中核は哲学側にあり、SEP / IEP / PhilPapers が主情報源になる。
`q-008` に PubMed を投げてもほぼ何も出ない。**PhilPapers は 403 で到達できなかった。**

**`article_type` を必ず本文に出す。** Review と原著と症例報告を同じ顔で並べない。
実例として、会で「fMRIで二重乖離が示された」と紹介された Fedorenko 2024
（PMID 38898296）は **Review** であり一次研究ではなかった。一次研究は会の後に
PNAS で出ている（Kean et al. 2026, PMID 42406951）。

**検証パスは機械的に行う。** 最終稿の全 PMID をメタデータ照会で再取得し、
タイトル・n数・`article_type` の一致を確認する。「たぶん実在する」で通さない。

### 並列化の設計

**問いごとに subagent を分け、中央で検証する。**
`q-005` を1件だけパイロットして品質を確認してから、残り7件を並列展開した。
いきなり8件走らせていない。

**subagent の報告を信用しない。** 全208件の PMID を独立に照会して検証した。
検証自体も専任エージェントに委ねたが、**監査用に6件のタイトルを原文で報告させ、
自分の記録と照合した**（「検証者を誰が検証するのか」を空白にしないため）。

**結果:** 捏造ゼロ。指摘は `article_type` の副次タグ脱落7件のみで、
うち2件は**同じ論文を `q-001` と `q-002` で違う書き方をしていた内部不整合**だった。
並列化で起きうる典型的な劣化なので、次回も同じ検査を行うこと。

### q-010 を切り出した理由

`q-007`（「内言語」は何を指すか）の調査前に、**日本語の「内言語」と英語の
_inner speech_ が同じものを指すのかを確かめる必要があった**。
結果、指すものが違った。心理学・神経科学系の辞典は「言語以前の思考や概念」まで含めるが、
失語症学系は「言語心像を思い浮かべる機能」という狭い対象を指す。
**用語が乱立しているのではなく、同一の語が分野をまたいで別の対象を指していた。**

### 高次脳機能データベースの分類を作り直した（2026-07-19〜20）

**分類軸を DSM-5 の6神経認知領域＋半球離断に揃えた。**
それまでは機能軸5つ（記憶／言語／行為／認知）に**部位軸（前頭葉）が1つ混ざっていた**。
6カテゴリ50項目 → **8カテゴリ90項目**。

**半側空間無視を「複雑性注意」に置いた。** 文献上は決着していない
（注意ネットワーク説／表象説／多成分症候群説の3つが並立）。
厚労省の令和5年版ガイドラインが注意障害の下位に置いていることと、
注意ネットワーク説が現在の主流であることを根拠に主収載先を決めたが、
**項目本文に3つの立場を併記し、決着していないと明記した。**

**論争がある項目は両論併記にした。** 本人の指示による方針。
視覚失認の分類粒度、相貌失認の連合型か多感覚性か、Gerstmann症候群の単一性と局在、
病態失認と疾病否認、触覚失認と astereognosis、構成障害・着衣失行の独立性、
アパシーの帰属、保続の帰属、失語型分類そのもの。**単一の正解として書いていない。**

**古典的失語分類は「記述の語彙」として残し、病巣推定には使わないと明記した。**
初発脳卒中失語60例で病巣と型の対応が明確だったのは35%のみ（PMID 26713015）。
同じ研究で全失語33.3%・伝導失語1.7%であり、**現行が伝導失語を立てて全失語を
立てていなかったのは頻度の実態と逆行していた。**

**疾患名を冠した項目を症候記述に置き換えた。**
「アルツハイマー型認知症による記憶障害」→「内側側頭葉型健忘」。
疾患名を冠すると疾患数×症候数に発散し、所見から鑑別を引く逆引きにも使えない。

**Anton症候群を病態失認の下位型として位置づけた。**
Anton (1899) が概念化した実体を指すために Babinski (1914) が anosognosia を造語した、
という起源関係がある（PMID 31220837）。同列に置くのは論理的に整合しない。

**一覧の定義を `constants/higherBrainFunction.ts` に切り出し、
`__tests__/higherBrainFunction.test.ts` で実ファイルと機械的に突き合わせるようにした。**
`page.tsx` へのハードコードが原因で、孤児（`aphasia.md`）とリンク切れ（`compulsive-tool-use`、
`grasping`）が実際に発生していた。二重管理自体は残るが、ずれたらテストが落ちる。

**`content/neurology/Neuroanatomy/高次脳機能/`（62本）には手を付けていない。**
`app/neurology/*` が `redirect("/")` で表示を封鎖しており、本文も30本が空。
